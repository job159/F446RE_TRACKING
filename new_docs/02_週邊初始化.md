# 02 — 週邊初始化 (CubeMX 設定與原因)

本章解釋每個週邊的設定值、選擇原因,以及修改時的影響範圍。

設定位置:**main.c 的 MX_xxx_Init()**(由 CubeMX 產生)。
大部分改動建議透過 CubeMX UI 修改再 regenerate,避免手動改 User Code 區外的內容被覆蓋。

---

## 1. 系統時脈

位於 [main.c::SystemClock_Config()](../Core/Src/main.c)

| 參數 | 值 | 原因 |
|---|---|---|
| Oscillator | **HSI (16 MHz 內部 RC)** | 不需外部石英,板子直接用 |
| PLL_M | 16 | PLL 輸入 = 16/16 = 1 MHz |
| PLL_N | 336 | VCO = 1 × 336 = 336 MHz |
| PLL_P | /4 | SYSCLK = 336/4 = **84 MHz** |
| PLL_Q | 2 | USB/SDIO 時脈,本案未用 |
| AHB | /1 | HCLK = 84 MHz |
| APB1 | /2 | PCLK1 = 42 MHz,timer clk ×2 = 84 MHz |
| APB2 | /1 | PCLK2 = 84 MHz,timer clk ×1 = 84 MHz |
| Voltage Scale | Scale3 | 84 MHz 最省電的電壓設定 |
| Flash latency | 2 WS | 84 MHz @ 2.7-3.6V 所需 |

**為何選 84 MHz 而非 180 MHz 上限?**
- 低耗電 (Scale3 允許)
- Timer clock 84 MHz 對 1 kHz ~ 60 kHz step PWM 綽綽有餘
- ADC clock 經 /4 = 21 MHz,在 ADC 規格安全區間

## 2. ADC1 與 ADC2

位於 [main.c::MX_ADC1_Init()](../Core/Src/main.c:187) / [MX_ADC2_Init()](../Core/Src/main.c:256)

| 參數 | 值 | 原因 |
|---|---|---|
| **ClockPrescaler** | PCLK2 /4 = 21 MHz | ADC 最高 36 MHz,/4 穩定有餘 |
| **Resolution** | 12-bit | F446 原生,0~4095 |
| **ScanConvMode** | ENABLE | 一次掃多通道,配合 DMA |
| **ContinuousConvMode** | ENABLE | 自動連續轉換,不用 CPU 每次 trigger |
| **DMAContinuousRequests** | ENABLE | DMA 自動回填 |
| **DataAlign** | RIGHT | 自然數值 |
| **NbrOfConversion** | 2 | 每顆 ADC 掃 2 通道 |
| **SamplingTime** | **28 Cycles** | LDR 阻抗較高(數十 kΩ 量級),28 cycles 給足充電時間;3 cycles 會讀不穩 |
| **EOCSelection** | EOC_SEQ_CONV | 每完成整個 sequence 才觸發 EOC,配合 DMA |
| **ExternalTrig** | SOFTWARE_START | 由軟體 `HAL_ADC_Start_DMA` 啟動 |

### 通道設定

| ADC | Rank | Channel | Pin | LDR ch |
|---|---|---|---|---|
| ADC1 | 1 | IN13 | PC3 | ch0 (左上) |
| ADC1 | 2 | IN12 | PC2 | ch2 (右下) |
| ADC2 | 1 | IN14 | PC4 | ch1 (右上) |
| ADC2 | 2 | IN11 | PC1 | ch3 (左下) |

**為何雙 ADC 並行?**
F446 的 ADC1/ADC2 可同時啟動獨立轉換,4 通道讀完大約:
```
2 rank × (28 + 12) cycles × (1/21MHz) ≈ 3.8 µs
```
比單顆 ADC 掃 4 通道快一倍,有利短週期控制迴圈。

### CubeMX Rank 重複 slot 注意事項

CubeMX UI 在 ADC 設定留有「幽靈」slot(`NbrOfConversion=2` 但 .ioc 會多記 Channel-2 / Channel-3 一筆),導致 `MX_ADC1_Init` / `MX_ADC2_Init` 多生一次 `HAL_ADC_ConfigChannel`。
修法:[main.c](../Core/Src/main.c) `USER CODE BEGIN ADCx_Init 2` 區段再呼叫一次 `HAL_ADC_ConfigChannel` 把 Rank 2 的 sample time 蓋回 28 cycles,確保 4 路 LDR 對稱。

## 3. DMA (ADC 用)

| DMA | Stream | 週邊 | 模式 | 說明 |
|---|---|---|---|---|
| DMA2 | **Stream0** | ADC1 | Circular, HalfWord | 自動回填 `dma_buf1[]` |
| DMA2 | **Stream3** | ADC2 | Circular, HalfWord | 自動回填 `dma_buf2[]` |

兩個 DMA stream IRQ 在 `MX_DMA_Init` 有 enable,但在 [app_adc.c::start_adc_dma()](../Core/Src/App/app_adc.c) 手動 `__HAL_DMA_DISABLE_IT(DMA_IT_HT | DMA_IT_TC)` 關掉中斷 — circular 模式下不需要 HT/TC 中斷,不關會因 ADC 連續轉換太快導致 ISR 暴風吃滿 CPU。

### 本專案其他 UART / USART 不用 DMA
`huart2 / huart4 / huart5` 皆用 blocking `HAL_UART_Transmit`(資料量極小,開機 init 跟 100 ms 遙測)。
未來若要高速串口(>10 kB/s),再切換到 DMA 版本。

## 4. TIM1 / TIM3 (步進 STEP PWM)

TIM1 掛 APB2 (84 MHz),TIM3 掛 APB1 (42 MHz 但 APB1 divider ≥ 2 時 timer clock ×2 = 84 MHz)。
兩顆 timer 設定相同:

| 參數 | 值 | 原因 |
|---|---|---|
| **Prescaler** | 84-1 | timer input 84 MHz → counter **1 MHz** |
| **Period (ARR)** | 1000 | 初始值,runtime 會被 `set_step_freq()` 改寫 |
| **ClockDivision** | DIV1 | 不需額外分頻 |
| **CounterMode** | UP | 單方向計數 |
| **PWM1 mode, CH1** | active level HIGH | STEP 訊號預設 |
| **Pulse** | 500 (50%) | runtime 也會被改為 `(ARR+1)/2` |
| **AutoReloadPreload** | DISABLE | 設定 ARR 立即生效(配合 `GenerateEvent`) |

### counter clock 都是 1 MHz
- APB2 prescaler /1 → timer input = 84 MHz,÷84 = **1 MHz** counter
- APB1 prescaler /2 → APB1 bus 42 MHz,**timer clock ×2 = 84 MHz**,÷84 = **1 MHz** counter

`get_timer_clock()` 動態從 RCC 讀 bus clock,若 APB divider 被改仍能算對,不用手動維護。

### STEP 頻率計算
```
ARR = counter_clock / step_freq - 1
```
例:1 MHz / 5000 Hz = 200,ARR = 199,對應 5 kHz STEP 脈衝。

### 16-bit ARR 的上下限
- ARR ≥ 0,最高頻率 < 1 MHz(理論),實務 TMC2209 + ramp 60 kHz 已是極限
- ARR ≤ 65535,最低頻率 ≈ 16 Hz

## 5. UART4 / UART5 (TMC2209 控制)

| 參數 | 值 |
|---|---|
| BaudRate | **115200** |
| WordLength | 8-bit |
| StopBits | 1 |
| Parity | None |
| HwFlowCtl | None |
| OverSampling | 16 |
| Mode | TX/RX (Asynchronous) |

**Pin:** UART4 = PC10(TX) / PC11(RX),UART5 = PC12(TX) / PD2(RX)

### TMC2209 UART 要求
- 單線半雙工 (SingleWire):RX 與 TX 接同一條線,用 pull-up 電阻拉高
- TMC2209 支援 9600 ~ 500k baud,**115200 是穩定甜區**
- 本程式只做 write (不讀回),所以 RX 腳可以不接

若 driver 不穩(大機器電感干擾大),可降到 57600。

---

## 6. USART2 (log / 指令)

同樣 115200 8N1,連接 Nucleo 的 ST-Link VCP (PA2=TX, PA3=RX)。
在 PC 用 serial terminal (例如 PuTTY、TeraTerm、minicom、Arduino serial monitor)開 115200 即可與系統互動。

## 7. GPIO

| Pin | Mode | Pull | Speed |
|---|---|---|---|
| PC6, PC8, PC9 (DIR, EN) | OUTPUT_PP | NOPULL | LOW |
| PB8 (EN) | OUTPUT_PP | NOPULL | LOW |
| **PC13 (B1)** | **INPUT** | **PULLUP** | — |
| ADC pins (PC1, PC2, PC3, PC4) | ANALOG | NOPULL | — |

### 為何 DIR / EN 只用 LOW speed?
這些訊號切換頻率極低(模式改變時才一次),LOW speed(約 2 MHz slew)足夠,且可降低 EMI。

### EN 邏輯
TMC2209 的 EN 是**低電位 enable**:
- Write `GPIO_PIN_RESET` → 馬達通電、有扭力
- Write `GPIO_PIN_SET` → 馬達斷扭(軸可手轉)

### B1 按鈕為何 polling 不用中斷
EXTI + ISR 會碰到彈跳、跟主迴圈共用狀態的 race 問題。
現在主迴圈每圈呼叫 [app_main.c::poll_button()](../Core/Src/App/app_main.c) 讀 pin,配合 180 ms 軟體 debounce,**比中斷更簡單穩定**。
細節見 [08_串口指令.md](08_串口指令.md#板載按鈕-b1-pc13)。

## 要修改週邊設定的正確流程

1. 打開 `F446RE_TRACKING.ioc` (CubeMX)
2. 修改 pinout / timer prescaler / baudrate 等
3. 按 GENERATE CODE
4. 重新編譯

**手動改 main.c 的 MX_xxx_Init 會被 CubeMX 下次 regenerate 時覆蓋**,除非寫在 `USER CODE BEGIN/END` 區塊內。本專案目前在 `ADCx_Init 2` 區段有一個針對 rank 重複 slot 的補丁,參見 [第 2 節](#2-adc1-與-adc2) 的說明。
