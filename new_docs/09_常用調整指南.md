# 09 — 常用調整速查

> 「我想要做 X → 該改哪一行」的任務導向速查表。
>
> **9 成的調整在 [tracking_config.h](../Core/Inc/App/tracking_config.h)** 一個檔。
> 進階才需要改 .c。

---

## ADC / LDR

### Q1　LDR 接反向（亮時 ADC 值低）
[tracking_config.h](../Core/Inc/App/tracking_config.h)：
```c
#define ADC_INVERT  1     // 1 = 亮→大, 0 = 讀原始值
```

### Q2　LDR 讀值要更平滑（犧牲反應）
```c
#define ADC_LPF_ALPHA_NEW  3     // 預設 9；改小 = 更平滑
```

### Q3　LDR 讀值要更快反應
```c
#define ADC_LPF_ALPHA_NEW  19    // 改大 = 更快但更吵
```

### Q4　校正時間太短不準
```c
#define SYS_BOOT_CALIBRATION_MS  10000   // 預設 5000
```

### Q5　永遠 valid:0（明明有光源）
先看 `STATUS` 的 `total:` 與 `contrast:`。若確實太低：
```c
#define TRACK_VALID_TOTAL_MIN         60    // 預設 140
#define TRACK_DIRECTION_CONTRAST_MIN  15    // 預設 28
```
或檢查 LDR 接線是否斷。

### Q6　LDR 雜訊大，noise floor 太低一直誤觸發
```c
#define LDR_BASELINE_MARGIN  30    // 預設 10
#define LDR_MIN_NOISE_FLOOR  20    // 預設 6
```

## 馬達細分 / 速度

### Q7　改細分（例如改回 1/16）
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) `CHOPCONF_MRES_BITS`：
```c
#define CHOPCONF_MRES_BITS  (4UL << 24)   // MRES 值見下表
```
| 細分 | MRES |
|---|---|
| 1/256 | 0 |
| 1/128 | 1 |
| 1/64  | 2 |
| **1/32 (預設)** | **3** |
| 1/16  | 4 |
| 1/8   | 5 |
| 1/4   | 6 |

⚠️ 改完細分：(1) STEP 頻率對應的轉速會變，(2) 軟限位 `M*_LIMIT_STEPS` 要重新換算。例如 1/32 → 1/16 時，同樣機構角度的 step 數剩一半。

### Q8　改 Manual 速度表（7 段）
[motor_control.c](../Core/Src/App/motor_control.c) 第 9-12 行：
```c
static const uint16_t MANUAL_SPEED_TABLE[TMC_SPEED_STAGE_COUNT] = {
   50, 100, 200, 400, 600, 1000, 5000,   /* F1..F7 */
   50, 100, 200, 400, 600, 1000, 5000    /* R1..R7 */
};
```
直接改 7 個數字，正反轉務必一致。若要改段數，同步改 [stepper_tmc2209.h](../Core/Inc/App/stepper_tmc2209.h) 的 `TMC_SPEED_STAGE_COUNT` 與 `TMC_DIR_SPLIT_STAGE`。

### Q9　增加電流（扭力不夠）
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) `VAL_IHOLD_IRUN`：
```c
#define VAL_IHOLD_IRUN  ((4UL << 16) | (24UL << 8) | 6UL)
                                       ^^  改成 0~31 的 IRUN
```
| IRUN | 約峰值電流 (Rsense 110 mΩ) |
|---|---|
| 16 (預設) | ≈ 0.89 A |
| 20 | ≈ 1.10 A |
| 24 | ≈ 1.30 A |
| 31 | ≈ 1.65 A |

⚠️ 超過 24 建議加散熱片。

### Q10　切 SpreadCycle（大扭力 / 較吵）
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) `VAL_GCONF`：
```c
#define VAL_GCONF  ((1UL << 6) | (1UL << 7) | (1UL << 2))   // 加 bit2 en_SpreadCycle
```

### Q11　反轉方向錯（只想翻一個軸）
TRACKING 模式：改 [tracking_config.h](../Core/Inc/App/tracking_config.h)：
```c
#define M1_TRACK_DIR   (-1)   // 只影響 TRACKING
```
Manual 模式不受 `M*_TRACK_DIR` 影響；要改 Manual 方向，可改 [motor_control.c::stage_to_signed()](../Core/Src/App/motor_control.c) 或對調接線。

### Q12　加減速太突兀
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c)：
```c
#define RAMP_STEP_HZ   400   // 預設 800，改小 = 更平順但啟停慢
#define RAMP_DELAY_MS  2     // 預設 1，改大 = 更平順
```

### Q13　加減速太慢（浪費時間）
```c
#define RAMP_STEP_HZ   2000  // 預設 800
#define RAMP_DELAY_MS  0     // 不延遲；只建議負載輕時
```

### Q14　IDLE 時馬達完全斷扭（可手轉軸）
[motor_control.c::MotorControl_StopAll()](../Core/Src/App/motor_control.c) 尾端加：
```c
HAL_GPIO_WritePin(AXIS1_EN_PORT, AXIS1_EN_PIN, GPIO_PIN_SET);
HAL_GPIO_WritePin(AXIS2_EN_PORT, AXIS2_EN_PIN, GPIO_PIN_SET);
```
要記得在 `MotorControl_ApplyCommand` / `SetManualStage` 進入時拉回 RESET，否則下次動不起來。

## 控制律（純 P，不是 PID）

> 本系統控制律是 **P-only**，沒有 KI / KD / RATE_LIMIT / integrator。
> 詳見 [06_PID_控制邏輯.md](06_PID_控制邏輯.md)。

### Q15　整體追蹤太慢
放大三段 KP：
```c
#define M1_KP_SMALL    150.0f   // 預設 100
#define M1_KP_MEDIUM   600.0f   // 預設 400
#define M1_KP_LARGE   1200.0f   // 預設 800
```

### Q16　追蹤會 overshoot / 共振 / 抖
1. **先降 KP_LARGE** 約 30 %。
2. 若還抖，降 `OUTPUT_GAIN`（例如 1.0 → 0.8）。
3. 或放大死區：`PID_ERR_DEADBAND 0.020 → 0.030`。

### Q17　兩軸不對稱（一軸快、一軸慢）
兩軸 KP 獨立，分開調。例如讓 M1 慢一點：
```c
#define M1_KP_SMALL     80.0f
#define M1_KP_MEDIUM   320.0f
#define M1_KP_LARGE    640.0f
```

### Q18　追到目標附近會「滴答」抖
放大死區：
```c
#define PID_ERR_DEADBAND  0.030f    // 預設 0.020
```

### Q19　目標明明在中央卻一直微調
死區太小或 LDR 雜訊太大。兩個選項：
- 放大 `PID_ERR_DEADBAND`
- 放大 `LDR_BASELINE_MARGIN` 讓 noise 無法跨越 floor

### Q20　大誤差時太衝
- 降 `KP_LARGE`
- 降 `OUTPUT_GAIN`
- 降 `MAX_STEP_HZ`
- 或降 `TRACK_ERR_CAP`（從 0.7 → 0.5，夾得更狠）

### Q21　小誤差時太鈍
- 提高 `KP_SMALL`
- 縮小 `PID_ERR_DEADBAND`

### Q22　機構左右不對稱
調整方向倍率：
```c
#define M1_POS_SCALE  1.20f   // 正轉更強
#define M1_NEG_SCALE  1.05f   // 反轉略減
```

### Q23　改控制週期
只支援 1 / 2 / 5 ms。串口即時：
```
PERIOD 2
```
或改 [tracking_config.h](../Core/Inc/App/tracking_config.h) 的 `SYS_CONTROL_PERIOD_MS` 改開機預設。
其他值送入會回 `ERR period only 1|2|5`。

## 軟限位

### Q24　調整可動角度（軟限位）
[tracking_config.h](../Core/Inc/App/tracking_config.h)：
```c
#define M1_LIMIT_STEPS   800     // ±45° (1:1 齒比, 1/32 microstep)
#define M2_LIMIT_STEPS   9600    // ±180° (1:3 齒比)
```
換算：`LIMIT_STEPS = 每圈步數 × 齒比 × (角度/360°)`。
每圈步數 = 200 × microstep（1/32 → 6400）。

### Q25　關閉軟限位（測試用）
```c
#define M1_LIMIT_ENABLE   0
#define M2_LIMIT_ENABLE   0
```

### Q26　歸零機構位置
串口送：
```
HOME
```
把位置累計器 `axis*_position_steps` 強制設為 0。
⚠️ 必須在機構實際位於中位時才送，否則軟限位會偏掉。

## 系統行為

### Q27　開機直接進 MANUAL 不追蹤
[app_main.c::AppMain_Init()](../Core/Src/App/app_main.c) 找到：
```c
g.mode_after_cal = MODE_TRACKING;
```
改成：
```c
g.mode_after_cal = MODE_MANUAL;
```

### Q28　略過校正直接動（不建議）
縮短校正：
```c
#define SYS_BOOT_CALIBRATION_MS  500   // 0.5 秒
```
完全略過會讓 baseline 全 0、delta 變 raw，LDR 弱光辨識失靈。

### Q29　改 ST-Link VCP baud
1. CubeMX 開 USART2 改 baud。
2. Generate code。
3. PC 端 serial monitor 同步改。

### Q30　加新串口指令
1. [tracking_types.h](../Core/Inc/App/tracking_types.h) 加 `SerialCmdId_t` enum。
2. [serial_cmd.c::parse_line()](../Core/Src/App/serial_cmd.c) 加 `if (!strcmp(txt, "XXX")) { enqueue(h, SERIAL_CMD_XXX, 0); return; }`。
3. [app_main.c::handle_cmd()](../Core/Src/App/app_main.c) 加 case 處理。

### Q31　改遙測週期
```c
#define SYS_TELEMETRY_PERIOD_MS  500    // 預設 100
```

### Q32　完全關掉遙測
[app_main.c::AppMain_Task()](../Core/Src/App/app_main.c) 最後一行註解掉：
```c
// Telemetry_Task(&g.telem, &g.snap);
```

### Q33　按鈕 cycle 太快（連點）
[app_main.c](../Core/Src/App/app_main.c)：
```c
#define BUTTON_DEBOUNCE_MS  300   // 預設 180
```

## 加新硬體

### Q34　加 LED 指示 valid 狀態
1. CubeMX 設 GPIO Output（例如 PA4）。
2. [app_main.c::run_control()](../Core/Src/App/app_main.c) 開頭加：
   ```c
   HAL_GPIO_WritePin(GPIOA, GPIO_PIN_4,
       g.ldr.frame.is_valid ? GPIO_PIN_SET : GPIO_PIN_RESET);
   ```

### Q35　加蜂鳴器告警（失去光源）
1. CubeMX 設 GPIO 或 PWM。
2. 在 run_control() 偵測 `is_valid` 從 1 變 0 的邊緣時 toggle。

### Q36　加 OLED (I2C)
1. CubeMX 啟 I2C1，Generate。
2. 引入 SSD1306 開源驅動。
3. 在 telemetry 旁邊新增顯示 task。

### Q37　加 RTC
1. CubeMX 啟 RTC + LSE。
2. 新檔 `app_rtc.c` 包 HAL_RTC_*。
3. 加串口指令查 / 設時間。

## 除錯

### Q38　程式卡在 `Error_Handler()`
- 接 SWD 設斷點，看 call stack。
- 通常是 ADC / UART init 失敗。
- 檢查 CubeMX clock 設定 & VMOT 是否上電。

### Q39　馬達不動
1. Manual 試 `F3`；雙軸都不動 → EN pin 是否拉低？UART4/5 init 是否成功？
2. 用邏輯分析儀看 STEP 波形是否出現。

### Q40　ADC 全 0 或全 4095
- 檢查 `ADC_INVERT` 值。
- 萬用表量 ADC pin 電壓，確認分壓正常。
- 看 `STATUS` 的 `adc:` 欄。

### Q41　Serial 沒反應
- baud 115200 8N1 / 對應 ST-Link VCP COM port。
- 送 `HELP` 測試。
- 檢查 `huart2` 是否 init 成功。
