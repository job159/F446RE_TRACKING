# 09 — 常用調整指南 (任務導向)

「我想要做某某效果 → 該改哪一行」的速查表。

> 💡 **9 成的調整都在 [tracking_config.h](../Core/Inc/App/tracking_config.h)** 一個檔。

---

## 🎯 ADC / LDR 相關

### Q1: LDR 接反方向了(亮時 ADC 值低)
[tracking_config.h](../Core/Inc/App/tracking_config.h):
```c
#define ADC_INVERT  1     // 0 = 不反 / 1 = 反
```

### Q2: 想讓 LDR 讀值更平滑(犧牲反應速度)
```c
#define ADC_LPF_ALPHA_NEW  3   // 預設 9; 改小 = 更平滑
```

### Q3: 想讓 LDR 讀值反應更快
```c
#define ADC_LPF_ALPHA_NEW  19   // 改大 = 更快但更吵
```

### Q4: 校正太短不準
```c
#define SYS_BOOT_CALIBRATION_MS  10000   // 預設 5000,改長
```

### Q5: 校正後永遠 valid:0(明明有光源)
1. 先看 `STATUS` 的 `total:` 值,可能是門檻太高
2. 降低 [tracking_config.h](../Core/Inc/App/tracking_config.h):
   ```c
   #define TRACK_VALID_TOTAL_MIN         60    // 預設 140
   #define TRACK_DIRECTION_CONTRAST_MIN  15    // 預設 28
   ```
3. 或檢查 LDR 接線是否斷

### Q6: LDR 雜訊大,校正後 noise floor 太低
```c
#define LDR_BASELINE_MARGIN  30    // 預設 10
#define LDR_MIN_NOISE_FLOOR  20    // 預設 6
```

## 🔄 馬達細分 / 速度

### Q7: 想改細分(例如改回 1/16)
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) **第 28 行**:
```c
#define CHOPCONF_MRES_BITS  (4UL << 24)   // MRES 值見下表
```
| 細分 | MRES |
|---|---|
| 1/16 | 4 |
| **1/32** | **3** (目前) |
| 1/64 | 2 |
| 1/128 | 1 |
| 1/256 | 0 |

⚠️ 改完細分,STEP 頻率對應的轉速會變,可能需要把速度表 ×2 或 /2。

### Q8: 想改 manual 速度的 7 個值
[motor_control.c](../Core/Src/App/motor_control.c) **第 9-13 行**:
```c
static const uint16_t MANUAL_SPEED_TABLE[14] = {
   50, 100, 200, 400, 600, 1000, 5000,  /* F1..F7 */
   50, 100, 200, 400, 600, 1000, 5000   /* R1..R7 */
};
```
直接改 7 個數字。**正反轉表須一致**(否則同一段 F/R 速度不對稱)。

### Q9: 想增加電流(扭力不夠)
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) **第 19 行**:
```c
#define VAL_IHOLD_IRUN  ((4UL << 16) | (24UL << 8) | 6UL)
                                       ^^ 改成 0~31
```

| IRUN | 約峰值電流(110mΩ Rsense) |
|---|---|
| 16 (預設) | ~0.89 A |
| 20 | ~1.10 A |
| 24 | ~1.30 A |
| 31 | ~1.65 A |

⚠️ 提高超過 24 建議加散熱。

### Q10: 想要 SpreadCycle (大扭力 / 較吵)
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) **第 16 行**:
```c
#define VAL_GCONF  ((1UL << 6) | (1UL << 7) | (1UL << 2))   // 加 bit2
```

### Q11: 反轉方向錯了(只想翻一個軸)
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) **第 16 行**(僅該軸的 driver,但本系統兩軸共用 GCONF...):
較好做法是改 [motor_control.c::stage_to_signed()](../Core/Src/App/motor_control.c:30):
```c
return (stage >= TMC_DIR_SPLIT_STAGE) ? hz : -hz;   // 對調 + / -
```
或對調機械接線。

### Q12: 加減速太突兀
[stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) **第 36-38 行**:
```c
#define RAMP_STEP_HZ   400   // 預設 800,改小 = 更平順但啟停慢
#define RAMP_DELAY_MS  2     // 預設 1,改大 = 更平順
```

### Q13: 加減速太慢 (浪費時間)
```c
#define RAMP_STEP_HZ   2000  // 預設 800,改大 = 更快
#define RAMP_DELAY_MS  0     // 完全不延遲
```

### Q14: 想 IDLE 時馬達完全斷扭(可手動轉軸)
[motor_control.c::MotorControl_StopAll()](../Core/Src/App/motor_control.c:55) 加兩行:
```c
HAL_GPIO_WritePin(AXIS1_EN_PORT, AXIS1_EN_PIN, GPIO_PIN_SET);
HAL_GPIO_WritePin(AXIS2_EN_PORT, AXIS2_EN_PIN, GPIO_PIN_SET);
```
要記得進入 TRACKING/MANUAL 時先重新 enable(現在 `StepperTmc2209_Init` 會做,但不重新呼叫)。
更乾淨:在 `MotorControl_ApplyCommand` / `SetManualStage` 開頭都拉低 EN。

## 🎚️ PID 控制

### Q15: 整體追蹤太慢
增加所有 KP(以 Motor1 為例):
```c
#define M1_KP_SMALL    150.0f   // 90 → 150
#define M1_KP_MEDIUM   300.0f   // 180 → 300
#define M1_KP_LARGE    500.0f   // 310 → 500
```

### Q16: 整體追蹤會 overshoot / 振盪
1. 先降 KP_LARGE 30%
2. 加一點 KD
3. 提高 RATE_LIMIT_HZ(若是因為 PID 變化太快被限制造成 jitter)

### Q17: Motor1 太慢、Motor2 太快(機構不對稱)
分開調整。把 Motor1 的 PID 全套放大,Motor2 的縮小:
```c
#define M1_KP_SMALL    140.0f   // 加快
#define M2_KP_SMALL    120.0f   // 放慢
```

### Q18: 追到目標附近會「滴答」抖
增加死區:
```c
#define PID_ERR_DEADBAND  0.030f   // 預設 0.015,放大兩倍
```

### Q19: 目標明明在中央卻一直微調
可能 KI 累積過頭:
```c
#define M1_KI  2.0f    // 預設 4,降一半
#define M2_KI  4.0f    // 預設 8,降一半
```

或調大死區衰減速度:
```c
#define PID_INTEGRATOR_DECAY  0.5f    // 預設 0.8,改小 = 衰減更快
```

### Q20: 大誤差時太衝
- 降 `KP_LARGE`
- 降 `RATE_LIMIT_HZ`
- 降 `OUTPUT_GAIN`

### Q21: 小誤差時太鈍
- 提高 `KP_SMALL`
- 縮小 `PID_ERR_DEADBAND`

### Q22: 想要 PID 完全不積分(純 PD 控制)
```c
#define M1_KI  0.0f
#define M2_KI  0.0f
```

### Q23: 控制週期太頻繁讓 CPU 吃力
目前預設已經是 5 ms (在 [tracking_config.h](../Core/Inc/App/tracking_config.h) `SYS_CONTROL_PERIOD_MS`)。
若想試更快反應,串口輸入 `PERIOD 1`(1 ms),但 stepper ramp 可能吃光 CPU。
若想更省 CPU,目前沒比 5 ms 更慢的選項,需自行加到 [serial_cmd.c::parse_line()](../Core/Src/App/serial_cmd.c) 的合法值表。

## 🐛 除錯

### Q35: 程式跑到 `Error_Handler()` 卡死
- 接 SWD 用 ST-Link debugger 設斷點
- 看 `__disable_irq()` 之前的 stack frame
- 通常是 ADC/UART 初始化失敗,檢查 CubeMX clock 設定

### Q36: 馬達不動 / 動作怪
1. 先用 manual 試:`F3` 看雙軸是否轉
2. 不轉 → 檢查 EN pin (拉低?)、UART4/5 是否成功初始化
3. 動作怪 → 用邏輯分析儀看 STEP 頻率是否符合預期

### Q37: ADC 全 0 或全 4095
- 檢查 `ADC_INVERT` 設定
- 檢查 LDR 分壓電路是否上電
- 用萬用電表量 ADC pin 電壓
- 啟動時 baseline 抓到值表示 ADC 在運作

### Q38: serial 沒回應
- 確認 baud 115200 8N1
- 確認 PC 端用 ST-Link VCP 對應的 COM port
- 試送 `HELP` 換行
- 檢查 huart2 在 main.c 是否有初始化
