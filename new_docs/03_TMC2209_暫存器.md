# 03 — TMC2209 暫存器設定

TMC2209 是 Trinamic 的靜音步進驅動 IC。本系統透過 **UART 單線模式**寫入 4 個關鍵暫存器,然後靠 STEP/DIR 腳位產生實際動作。

所有設定在 [stepper_tmc2209.c](../Core/Src/App/stepper_tmc2209.c) 最上方集中,改值改完就 rebuild。

## UART Frame 格式

TMC2209 write 指令固定 8 bytes:

| byte | 值 | 意義 |
|---|---|---|
| 0 | 0x05 | SYNC (固定) |
| 1 | slave_addr | 0x00~0x03,由 MS1/MS2 腳位硬體決定 |
| 2 | reg \| 0x80 | 要寫的暫存器位址,bit7=1 表示 write |
| 3 | val[31:24] | Big-endian 資料 |
| 4 | val[23:16] | |
| 5 | val[15:8]  | |
| 6 | val[7:0]   | |
| 7 | CRC8 | polynomial 0x07 |

程式對應 [stepper_tmc2209.c::write_reg()](../Core/Src/App/stepper_tmc2209.c:58) 與 CRC 函式。

## 開機初始化順序

每次 `StepperTmc2209_Init` 會:

1. EN 拉高(disable 馬達)
2. DIR 設 forward
3. 延遲 2ms 讓 driver 醒來
4. 寫 **GCONF** → 啟用 UART 控制模式
5. 寫 **IHOLD_IRUN** → 設定運轉/待機電流
6. 寫 **CHOPCONF** → 設定細分 (microstep) + 關斷特性
7. 寫 **PWMCONF** → StealthChop 靜音模式參數
8. EN 拉低(enable 馬達)
9. 啟動 STEP PWM,預設 stage 0 (最慢 50 Hz)

**每個 write 後 1ms delay**,給 TMC 處理時間。

## 1. GCONF (0x00) — 全域設定

目前設定值:

```c
#define VAL_GCONF  ((1UL << 6) | (1UL << 7))  // = 0x000000C0
```

對應的 bit:

| bit | 名稱 | 設 1 的意義 |
|---|---|---|
| 6 | **pdn_disable** | 關閉 PDN pin 的自動 power-down,允許 UART 永遠通 |
| 7 | **mstep_reg_select** | 使用 CHOPCONF 的 MRES 欄位決定細分,忽略 MS1/MS2 硬體 pin |

**為何設這兩個 bit?**
- 沒 pdn_disable:每次 UART 寫入時 driver 會誤判 PDN 要 power-down,造成異常。
- 沒 mstep_reg_select:細分仍由 MS1/MS2 腳決定,程式改 MRES 不會生效。

### 其他可用 bit(目前未啟用)
| bit | 名稱 | 若設 1 |
|---|---|---|
| 0 | I_scale_analog | 用 VREF 腳類比電流控制 (目前用 IHOLD_IRUN 數位設定) |
| 1 | internal_Rsense | 用內建參考電阻(一般外掛 110mΩ 就夠) |
| 2 | **en_SpreadCycle** | **從 StealthChop 切到 SpreadCycle (大扭力但吵)** |
| 3 | shaft | 反轉預設旋轉方向 |
| 4 | index_otpw | 用 INDEX 腳輸出過溫警告 |
| 8 | multistep_filt | 較平滑但較慢的步進響應 |

想要**高扭力追蹤**可以 OR 進 `(1UL << 2)`,但會變吵。

## 2. IHOLD_IRUN (0x10) — 電流設定

目前設定:

```c
#define VAL_IHOLD_IRUN  ((4UL << 16) | (16UL << 8) | 6UL)  // = 0x00041006
```

| bits | 欄位 | 本值 | 意義 |
|---|---|---|---|
| [4:0]   | **IHOLD**      | 6  | 靜止保持電流(0~31) |
| [12:8]  | **IRUN**       | 16 | 運轉電流(0~31) |
| [19:16] | **IHOLDDELAY** | 4  | 從 IRUN 降到 IHOLD 的過渡延遲(0~15) |

**電流計算公式** (With Rsense = 110mΩ, VREF = 2.5V):

```
I_peak (A) = (IRUN + 1)/32 × (V_fs / (R_sense + 20mΩ)) × 1/√2
```

對 IRUN=16 → 約 I_peak ≈ 0.89 A (RMS 約 0.63 A),適合 17HS 系列小機。

### 想改電流?

| 馬達規格 | 建議 IRUN |
|---|---|
| 17HS15 (小,1 A) | 14~18 |
| 17HS19 (中) | 20~24 |
| 23HS22 (大,2 A) | 26~31 |

**注意**:電流越大越吵、越燙,超過 24 建議加散熱片。
IHOLD 建議為 IRUN 的 1/2 ~ 1/3,節能又不會失鎖。

## 3. CHOPCONF (0x6C) — 斬波 & 細分

目前設定(`VAL_CHOPCONF`):

```c
CHOPCONF_BASE     = 0x10000053
CHOPCONF_MRES_BITS = (3UL << 24)   // 1/32 細分
VAL_CHOPCONF      = 0x13000053
```

展開:

| bits | 欄位 | 本值 | 意義 |
|---|---|---|---|
| [3:0]   | **TOFF**    | 3  | 斬波 off time,3 = 關閉模式? 其實 3 是最小有效值之一 |
| [6:4]   | **HSTRT**   | 5  | Hysteresis start |
| [10:7]  | **HEND**    | 0  | Hysteresis end |
| [16:15] | **TBL**     | 2  | Blank time(抗干擾窗) |
| [27:24] | **MRES**    | 3  | **微步細分** → 1/32 |
| [28]    | INTPOL      | 0  | 不做 256 內插(若要更平滑設 1) |
| [30]    | DISS2G      | 0  | 保留預設 |

### MRES 對照表(關鍵!)

```
MRES 值   細分         每圈 step 數(1.8°馬達)
 0      1/256          51200
 1      1/128          25600
 2      1/64           12800
 3      1/32            6400   ← 目前設定
 4      1/16            3200
 5      1/8             1600
 6      1/4              800
 7      1/2              400
 8      full step        200
```

### 細分怎麼選?
- **細分越高** → 越平滑、扭力下降、需要更高 STEP 頻率
- **細分越低** → 振動明顯但扭力保留

一般追蹤系統建議 **1/16 ~ 1/32**,本專案選 1/32 兼顧安靜與解析度。

### 想改細分?
只改一個數字:[stepper_tmc2209.c:28](../Core/Src/App/stepper_tmc2209.c:28)
```c
#define CHOPCONF_MRES_BITS   (3UL << 24)   // 把 3 改成上表中的 MRES 值
```

同時要考慮 **STEP 頻率要對應調整**,否則轉速會變:
- 從 1/32 改回 1/16,要讓相同轉速,STEP 頻率要 ×1/2

## 4. PWMCONF (0x70) — StealthChop 靜音模式

目前設定:

```c
#define VAL_PWMCONF  0xC10D0024
```

展開(這組是 Trinamic datasheet 建議的**萬用預設值**,一般不需改):

| bits | 欄位 | 本值 |
|---|---|---|
| [7:0]   | PWM_OFS   | 0x24 (36)  |
| [15:8]  | PWM_GRAD  | 0x0D (13)  |
| [17:16] | PWM_FREQ  | 0x01 (2/1024 fCLK) |
| [18]    | PWM_AUTOSCALE | 1 (自動調整) |
| [19]    | PWM_AUTOGRAD  | 1 (自動調整 gradient) |
| [21:20] | FREEWHEEL | 0 (停機輸出 normal) |
| [27:24] | PWM_REG   | 0x0C (reg 係數) |
| [31:28] | PWM_LIM   | 0xC (上限) |

### 想要更吵但更有力?
改成 **SpreadCycle**:在 GCONF bit2 設 1,忽略 PWMCONF。

### 想要更安靜?
把 PWM_AUTOSCALE / PWM_AUTOGRAD 開著(目前已開),讓 TMC 自動學習。
剛開機前幾轉會有微調聲音,之後就穩定。

## 暫存器寫入後如何驗證?

TMC2209 支援 read(透過單線 UART 讀回),但本程式**目前只做 write**。
若需要驗證,可以接邏輯分析儀在 STEP/DIR 觀察訊號,或用手感受扭力。

若要加 read 功能,可以參考 Trinamic datasheet section 5.5.1,在 `stepper_tmc2209.c` 增加 `read_reg()` 函式。

## 常見故障對照

| 症狀 | 可能原因 | 修正 |
|---|---|---|
| 馬達完全不動 | EN 沒拉低 / 沒收到 STEP | 檢查 EN pin、TIM 是否啟動 |
| 馬達抖動不走 | UART 沒寫入成功 / 接線錯 | 檢查 UART TX 腳、baud、CRC |
| 走幾步就卡住 | 電流太小 / 負載過重 | 提高 IRUN,確認 VMOT 電壓 |
| 非常吵 | 在 SpreadCycle 模式 | GCONF bit2 設 0 切回 StealthChop |
| 反轉方向錯 | 佈線或機械 | GCONF bit3 (shaft) 設 1 翻轉,或對調 A1/A2 線 |
| 高速失步 | STEP 頻率超過 ramp 能力 | 降低 max_hz 或加 ramp 時間 |
