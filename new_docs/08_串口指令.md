# 08 — 串口指令與遙測格式

## 連線設定

| 項目 | 值 |
|---|---|
| 連接埠 | Nucleo ST-Link VCP (USB → COMx) |
| Baud rate | 115200 |
| Data | 8 bits |
| Parity | None |
| Stop | 1 |
| Flow control | None |
| Line ending | CR or LF (兩者都接受) |

PC 端工具:PuTTY / TeraTerm / minicom / Arduino IDE Serial Monitor / VS Code Serial Monitor。

## 指令大寫小寫不分

`track`、`TRACK`、`Track` 都接受。
前後空白會被去掉。

---

## 模式切換

| 指令 | 簡寫 | 行為 |
|---|---|---|
| `IDLE`   | `0` | 停止所有馬達,等指令 |
| `TRACK`  | `1` | 進入自動追蹤(需先校正完成) |
| `MANUAL` | `2` | 進入手動模式(需先校正完成) |

> 校正中送指令會**排隊**,校正結束才執行。

## 校正

| 指令 | 行為 |
|---|---|
| `RECAL` | 立刻重新進入校正,持續 5 秒(`SYS_BOOT_CALIBRATION_MS`) |

## 手動 stage

每次發指令都會立刻切換速度;若不在 MANUAL 模式會自動進入。

| 指令 | 等同 | 馬達動作 |
|---|---|---|
| `F1` | `MAN 1`  | 雙軸正轉 50 Hz |
| `F2` | `MAN 2`  | 雙軸正轉 100 Hz |
| `F3` | `MAN 3`  | 雙軸正轉 200 Hz |
| `F4` | `MAN 4`  | 雙軸正轉 400 Hz |
| `F5` | `MAN 5`  | 雙軸正轉 600 Hz |
| `F6` | `MAN 6`  | 雙軸正轉 1000 Hz |
| `F7` | `MAN 7`  | 雙軸正轉 5000 Hz |
| `R1`..`R7` | `MAN 8`..`MAN 14` | 雙軸反轉,速度同上 |

> Manual 模式**兩軸同步**動作,因為共用同一個 stage。
> 想要兩軸獨立的 manual,需要修改 [manual_control.c](../Core/Src/App/manual_control.c) 與 [serial_cmd.c](../Core/Src/App/serial_cmd.c) 增加 axis 引數。

## 控制週期

| 指令 | 結果 |
|---|---|
| `PERIOD 1` | 1 ms (最快反應,但 stepper ramp 會吃光 CPU) |
| `PERIOD 2` | 2 ms |
| `PERIOD 5` | **5 ms (預設,也是建議值)** |

只允許 1 / 2 / 5,其他值回 `ERR period only 1|2|5`。
預設值在 [tracking_config.h](../Core/Inc/App/tracking_config.h) `SYS_CONTROL_PERIOD_MS`。

## 查詢

| 指令 | 回應範例 |
|---|---|
| `STATUS` | `STATUS mode:TRACK idle:- cal:1 valid:1 total:340 contrast:120 cmd:6200,-510` |
| `CAL?` | `CALDATA base:1234,1230,1240,1235 floor:18,16,20,15 done:1` |
| `CFG?` | `CONFIG period_ms:1 track_min:140 contrast_min:28 M1_kp:90/180/310 M2_kp:180/360/620 invert:1` |
| `HELP` | 列出所有指令 |

## 板載按鈕 (B1, PC13)

按一次 = cycle 到下一個 manual stage(0 → 1 → ... → 13 → 0)。
debounce 180 ms,防止連跳。

**任何時候按下都會直接進 MANUAL 模式**(包含開機 5 秒校正期間),校正會被打斷、馬達立刻以該 stage 速度轉。
若要保留校正,請用串口指令 `F1..F7` / `R1..R7` / `MANUAL` 取代(串口指令會在校正中排隊,校正完才生效)。

GPIO 設定為 Input + Pull-up,polling 讀取(非中斷)。

## 遙測自動輸出

每 `SYS_TELEMETRY_PERIOD_MS` (預設 100 ms) 自動送一行:

```
123 mode:TRACK sub:- cal:1 valid:1 adc:200,1800,180,210 base:150,160,155,148 d:50,1640,25,62 err:843,-72 cmd:6200,-510 stg:255
```

### 欄位解讀

| 欄位 | 範例 | 意義 |
|---|---|---|
| `123` | seq 號 | 累計輸出計數,可確認沒掉行 |
| `mode:` | TRACK | 系統模式 (IDLE/TRACK/MANUAL) |
| `sub:`  | - / CAL / WAIT | IDLE 時的子狀態,其他模式為 `-` |
| `cal:`  | 0 / 1 | 校正完成 |
| `valid:` | 0 / 1 | 目前 LDR 訊號是否有效 |
| `adc:` | 200,1800,180,210 | 4 路經 LPF + 反向後的數值 |
| `base:` | 150,160,155,148 | 校正得到的 baseline |
| `d:` | 50,1640,25,62 | delta(超過 baseline+noise floor 的部分) |
| `err:` | 843,-72 | error_x, error_y 各 ×1000 顯示 |
| `cmd:` | 6200,-510 | 馬達 1 / 馬達 2 的 step Hz (帶正負方向) |
| `stg:` | 255 / 0..13 | manual stage,255 表示無效(非 manual 模式) |

### 想關掉遙測?
方法 A — 改成更慢:[tracking_config.h](../Core/Inc/App/tracking_config.h):
```c
#define SYS_TELEMETRY_PERIOD_MS  1000   // 1 秒一次
```

方法 B — 完全停:在 [app_main.c::AppMain_Task()](../Core/Src/App/app_main.c) 註解掉 `Telemetry_Task(...)`。

## 指令範例(典型操作)

### 開機後立刻測試
```
（等 5 秒校正自動完成）
STATUS              ← 確認 cal:1
F3                  ← 切到正轉 200 Hz 試試馬達
（聽到雙軸轉動）
IDLE                ← 停下
TRACK               ← 進追蹤模式
（用手電筒照 LDR）
STATUS              ← 看 valid:1 cmd:?,? 數值
```

### 重新校正 + 切手動
```
IDLE                ← 先回 IDLE
RECAL               ← 開始校正
F2                  ← 校正中發指令會被排隊
（等校正完）
（自動進 MANUAL 模式並執行 F2 = 100 Hz 正轉）
```

### 想要更快的控制週期(1 ms)
```
PERIOD 1
```
注意:stepper 的 ramp 在 1 ms 週期下可能吃光 CPU 造成 UART / 按鈕反應遲緩,建議維持預設 5 ms。

### 查目前所有重要設定
```
CFG?
```

## 自製 PC 端腳本範例 (Python)

```python
import serial, time
s = serial.Serial('COM5', 115200, timeout=1)

# 開機後等校正完
time.sleep(6)

# 發指令
s.write(b'TRACK\n')

# 持續讀遙測
while True:
    line = s.readline().decode(errors='ignore').strip()
    if line.startswith('STATUS') or 'mode:' in line:
        print(line)
```

## Bonus: 為什麼用 polling RX 而不是中斷 / DMA?

[serial_cmd.c::SerialCmd_PollRx()](../Core/Src/App/serial_cmd.c) 在主迴圈每次呼叫,直接讀 RXNE 旗標。
原因:
- 指令量小(人類打字速度,< 10 byte/s)
- polling 比中斷簡單,沒 race condition
- RXNE FIFO 1-byte,只要主迴圈跑得比 ~10 kHz 快(實際 100 kHz+)就不會掉字

若需要支援大量 RX (例如圖檔上傳),改用 `HAL_UART_Receive_DMA` + circular buffer。
