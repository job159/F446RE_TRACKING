# 07 — 馬達細分、速度表、加減速

## 1. 步進馬達基本

1.8° 步進馬達:
- **整步 (full step)**: 200 步 / 圈
- **n 細分**: 200 × n 步 / 圈

|微步細分 | 每圈 step | 1 step 角度 |
|---|---|---|
| Full   |   200 | 1.8° |
| 1/2    |   400 | 0.9° |
| 1/4    |   800 | 0.45° |
| 1/8    |  1600 | 0.225° |
| 1/16   |  3200 | 0.1125° |
| **1/32** | **6400** | **0.05625°** ← 本系統 |
| 1/64   | 12800 | 0.028° |
| 1/128  | 25600 | 0.014° |
| 1/256  | 51200 | 0.007° |

### 細分 vs 扭力 vs 平滑度

- **細分越高** → 越平滑、解析度越好
- **細分越高** → **微步扭力越低**(microstep torque),
  最差情況約 = 全步扭力 × sin(90°/n)
- 1/32 仍能保留 **約 99.5% 全步扭力**,實用甜區
- 1/256 微步扭力只剩 ~0.6%,只適合超精細無負載定位

## 3. STEP 頻率對應實際轉速

```
轉速 (rev/s) = STEP_freq / (200 × microstep)
轉速 (rpm)   = STEP_freq × 60 / (200 × microstep)
```

| STEP_freq | 1/16 細分 | **1/32 細分** | 1/64 細分 |
|---|---|---|---|
|    50 Hz |   0.94 rpm |   **0.47 rpm** |   0.23 rpm |
|   100 Hz |   1.88 rpm |   **0.94 rpm** |   0.47 rpm |
|   200 Hz |   3.75 rpm |   **1.88 rpm** |   0.94 rpm |
|   400 Hz |   7.50 rpm |   **3.75 rpm** |   1.88 rpm |
|   600 Hz |  11.25 rpm |   **5.63 rpm** |   2.81 rpm |
|  1000 Hz |  18.75 rpm |   **9.38 rpm** |   4.69 rpm |
|  5000 Hz |  93.75 rpm |  **46.88 rpm** |  23.44 rpm |

> **註**:細分翻倍 → 同 STEP 頻率下轉速減半。
> 想保持與 1/16 同樣轉速,STEP 頻率要 ×2。

### 想改速度?
直接改陣列數字,例如想要 7 段細微分布:
```c
static const uint16_t MANUAL_SPEED_TABLE[14] = {
   25, 50, 100, 200, 400, 800, 1600,
   25, 50, 100, 200, 400, 800, 1600
};
```

數量必須維持 14 個(7+7)。
若要改數量,還要改 [stepper_tmc2209.h](../Core/Inc/App/stepper_tmc2209.h):
```c
#define TMC_SPEED_STAGE_COUNT   14U   // 改成新總數
#define TMC_DIR_SPLIT_STAGE     7U    // 改成新「正轉段數」
```
並對應更新 serial_cmd.c 的 F/R 範圍判斷。

## 5. STEP 頻率上下限

### 下限
程式 `set_step_freq` 用 16-bit ARR:
```
ARR = counter_clk / hz - 1
```
TIM1 (APB2) 跟 TIM3 (APB1 ×2) 的 timer input 都是 84 MHz,Prescaler=83 → counter_clk = **1 MHz**。
ARR_max = 65535 → **最低 hz ≈ 16 Hz**

低於 16 Hz 程式會回 HAL_ERROR(實際上 50 Hz 是安全值)。

### 上限
程式檢查 `if (counter_clk <= hz) return HAL_ERROR;` → **最高 hz < 1 MHz**

但實務上 TMC2209 + ramp 在 60kHz 以上很容易失步,
所以 [tracking_config.h](../Core/Inc/App/tracking_config.h) 有:
```c
#define M1_MAX_STEP_HZ   60000U
#define M2_MAX_STEP_HZ   60000U
```
作為 PID 輸出上限。Manual 速度表也應該別超過 60kHz。

## 6. Ramp 加減速邏輯

直接從 0 跳到 5000 Hz 馬達會失步,所以 [stepper_tmc2209.c::ramp_freq()](../Core/Src/App/stepper_tmc2209.c:104) 漸進式調速。

```c
#define RAMP_STEP_HZ   800   /* 每次最多跳 800 Hz */
#define RAMP_DELAY_MS  1     /* 每次跳之間延遲 1ms */
#define DIR_SETTLE_MS  2     /* 改方向後給 driver 2ms 穩定 */
```

### 從 0 → 5000 過程
```
0 → 800 → 1600 → 2400 → 3200 → 4000 → 4800 → 5000
   ↑1ms   ↑1ms   ↑1ms   ↑1ms   ↑1ms   ↑1ms   ↑1ms
```
總時間 ≈ 7 ms。

### 反向切換
[stepper_tmc2209.c::apply_speed()](../Core/Src/App/stepper_tmc2209.c) 三分支:
1. 從 0 啟動 → 設方向 → 等 2ms → 直接設速度
2. 同方向 → 直接 ramp 到新速度
3. 反向 → ramp 到最低速 → 切方向 → 等 2ms → ramp 到新速度

反向最久可能 30~50 ms(看當前速度多高)。

### Ramp 太慢?
- 加大 `RAMP_STEP_HZ` (例如 2000),減少跳次
- 但太大會失步,2000 是常見上限
- 或減小 `RAMP_DELAY_MS` 到 0(立即連續設定),適合扭力夠的場景

### Ramp 太突兀?
- `RAMP_STEP_HZ` 改 200~400
- 非常順但啟停變慢

## 7. PID 模式下不需手動 ramp?

PID 輸出本身有 **RATE_LIMIT** 保護,所以即使 PID 輸出突然從 +0 變 +60000,
實際 `MotorControl_ApplyCommand` → `SetSignedHz` 仍會經 ramp 漸進。

雙保險:RATE_LIMIT 控制「PID 算出的 hz 變化速率」,RAMP 控制「實際送到 timer 的速率」。
RATE_LIMIT 較大時 RAMP 會接手平滑;RATE_LIMIT 很小時 PID 自己就慢慢走,RAMP 只做最後微調。

## 8. 高速能不能跳過 ramp?

可以。若機構慣量小、driver 餘裕大,可以把 `RAMP_STEP_HZ` 設成等於 `MAX_STEP_HZ`,等於完全不 ramp。
這對 PID 模式幾乎無感(因為 RATE_LIMIT 會做大部分工作)。

對 Manual 模式則會直接從靜止跳到指定速度,可能失步,**不建議**。

## 9. EN 控制 (扭力斷電)

```c
HAL_GPIO_WritePin(en_port, en_pin, GPIO_PIN_RESET);  // enable (有扭力)
HAL_GPIO_WritePin(en_port, en_pin, GPIO_PIN_SET);    // disable (軸可手轉)
```

目前 `MotorControl_StopAll()` **沒拉高 EN**,僅停 STEP。
也就是 IDLE 狀態馬達還是有保持扭力(消耗 IHOLD 電流)。

若想 IDLE 完全斷扭,在 [motor_control.c::MotorControl_StopAll()](../Core/Src/App/motor_control.c:55) 加:
```c
HAL_GPIO_WritePin(AXIS1_EN_PORT, AXIS1_EN_PIN, GPIO_PIN_SET);
HAL_GPIO_WritePin(AXIS2_EN_PORT, AXIS2_EN_PIN, GPIO_PIN_SET);
```
但要記得進入 TRACKING/MANUAL 時拉低。
